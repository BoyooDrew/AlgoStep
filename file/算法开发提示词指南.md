## 第一批：排序类（sort/）

### 1. 冒泡排序 Bubble Sort

**可视化引导：**
```
可视化方式：用水平柱状图展示一维数组，每个元素是一个竖直的柱子，高度代表数值大小。
- 比较时：当前比较的两个元素高亮为橙色，其他元素保持默认色
- 交换时：两个元素位置互换，用动画平滑移动
- 每轮结束：已排序的部分变为绿色，标记为"已排序"
- 步骤拆分：每一步只做一对元素的比较和可能的交换，让学生看清每次操作
- 输入：用户可自定义数组内容（逗号分隔的数字），或点击随机生成
- 难度标签：入门
```

**参考代码：**

```cpp
// 冒泡排序 - Bubble Sort
// 时间复杂度: O(n^2)  空间复杂度: O(1)
// 思想: 相邻元素两两比较，较大的元素逐步"冒泡"到末尾
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {       // 外层循环: 轮数(趟数 pass)
        for (int j = 0; j < n - 1 - i; j++) { // 内层循环: 两两比较
            if (arr[j] > arr[j + 1]) {         // 比较相邻元素
                int temp = arr[j];             // 交换 swap
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
        // 第 i 趟结束, arr[n-1-i] 已确定
    }
}
```

核心思路：相邻元素两两比较，每轮将当前未排序部分的最大值"冒泡"到末尾，经过 n-1 轮后数组有序。

**教学重点（你补充给AI）：** 重点展示相邻元素的两两比较和交换过程，让学生理解为什么每轮都能把最大值"浮"到末尾。

### 2. 选择排序 Selection Sort

**可视化引导：**
```
可视化方式：用水平柱状图展示一维数组，和冒泡排序保持一致的视觉风格。
- 每轮开始：标记当前轮次要找最小值的位置（用蓝色竖线指示）
- 遍历比较：逐个扫描未排序部分，当前比较元素高亮为橙色
- 找到最小值后：高亮为绿色，然后与未排序部分第一个元素交换位置
- 交换动画：两个元素平滑移动互换位置
- 已排序部分：标记为绿色"已排序"
- 步骤拆分：每一步要么是"比较当前元素和当前最小值"，要么是"交换最小值到正确位置"
- 难度标签：入门
```

**参考代码：**

```cpp
// 选择排序 - Selection Sort
// 时间复杂度: O(n^2)  空间复杂度: O(1)
// 思想: 每轮从未排序部分选出最小元素, 放到已排序部分末尾
void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {         // 外层循环: 轮数
        int minIdx = i;                        // 假设当前位置最小
        for (int j = i + 1; j < n; j++) {       // 在未排序部分找最小值
            if (arr[j] < arr[minIdx]) {
                minIdx = j;                     // 更新最小值索引
            }
        }
        if (minIdx != i) {                     // 交换最小值到正确位置
            int temp = arr[i];
            arr[i] = arr[minIdx];
            arr[minIdx] = temp;
        }
        // 第 i 轮结束, arr[0..i] 已排序
    }
}
```

核心思路：每轮从未排序部分找到最小元素，与未排序部分首元素交换，逐步扩大已排序区间。

### 3. 插入排序 Insertion Sort

**可视化引导：**
```
可视化方式：用水平柱状图展示一维数组，保持排序类的统一视觉风格。
- 当前待插入元素：高亮为紫色/靛蓝色，从已排序部分的右侧取出
- 比较过程：待插入元素逐个与左侧已排序元素比较，被比较元素高亮为橙色
- 插入操作：找到正确位置后，元素平滑移动插入，比它大的元素右移腾出空间
- 已排序部分：用浅色背景标记，随着每轮递增
- 步骤拆分：每一步是一次比较或一次插入移动，让学生看清"拿牌-比较-插入"的过程
- 难度标签：入门
```

**参考代码：**

```cpp
// 插入排序 - Insertion Sort
// 时间复杂度: O(n^2)  空间复杂度: O(1)
// 思想: 将未排序元素逐个插入到已排序部分的正确位置
void insertionSort(int arr[], int n) {
    for (int i = 1; i < n; i++) {              // 从第2个元素开始
        int key = arr[i];                       // 当前待插入元素 (key)
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {         // 找插入位置 (向前查找)
            arr[j + 1] = arr[j];                 // 元素后移
            j--;
        }
        arr[j + 1] = key;                       // 插入到正确位置
        // arr[0..i] 已排序
    }
}
```

核心思路：类似打牌时整理手牌，每次取一个未排序元素，在已排序部分中找到合适位置插入。

### 4. 快速排序 Quick Sort

**可视化引导：**
```
可视化方式：用水平柱状图展示一维数组，保持排序类统一风格。
- Pivot选择：用醒目颜色（金色）标记当前轮次的基准元素
- 分区过程：用颜色区分三个区域——小于Pivot（蓝色）、等于Pivot（金色）、大于Pivot（红色）
- 递归展示：每完成一次分区，子数组用方括号或背景色框住，标注递归深度
- 交换动画：元素位置互换时平滑移动
- 步骤拆分：选择Pivot → 逐个比较分区 → 标记分区结果 → 递归进入子数组
- 难度标签：基础
```

**参考代码：**

```cpp
// 快速排序 - Quick Sort
// 平均时间复杂度: O(n log n)  空间复杂度: O(log n)
// 思想: 分治法 - 选基准值(pivot), 将数组分为两部分, 递归排序
void quickSort(int arr[], int left, int right) {
    if (left >= right) return;                   // 递归终止条件 (base case)

    // 分区操作 partition
    int pivot = arr[right];                      // 基准值 pivot (选最右元素)
    int i = left;
    for (int j = left; j < right; j++) {        // 扫描元素
        if (arr[j] < pivot) {                    // 比基准值小 → 放左边
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
            i++;
        }
    }
    // pivot 归位
    int temp = arr[i];
    arr[i] = arr[right];
    arr[right] = temp;
    // 此时: arr[left..i-1] < pivot <= arr[i+1..right]

    quickSort(arr, left, i - 1);                 // 递归排序左半部分 (recursion)
    quickSort(arr, i + 1, right);                // 递归排序右半部分 (recursion)
}
```

核心思路：分治法（divide and conquer）——选择基准值（pivot），将数组分为"小于pivot"和"大于等于pivot"两部分，然后递归排序子数组。

**教学重点（你补充给AI）：** 这是第一个分治算法，重点是让学生理解"选基准→分区→递归"的整体思路，可视化要清晰展示分区后左右两组的边界。

## 第二批：搜索类（search/）

### 5. 二分查找 Binary Search

**可视化引导：**
```
可视化方式：用水平柱状图展示有序数组，每个元素标注索引值。
- 搜索区域：当前搜索范围用背景色标记，已排除的元素变灰
- 左右指针：用三角箭头标记 left 和 right 指针位置
- 中间元素：当前 mid 元素高亮为橙色
- 比较结果：显示"mid值与目标值的关系"（太大/太小/找到）
- 缩小范围：排除的元素逐个变灰，搜索范围收窄，动画要体现"排除一半"
- 找到目标：目标元素高亮为绿色 + 弹出动画
- 未找到：所有元素变灰，显示"未找到"
- 输入：用户输入目标值
- 难度标签：入门
```

**参考代码：**

```cpp
// 二分查找 - Binary Search
// 时间复杂度: O(log n)
// 前提: 数组必须是有序的 (sorted array)
// 思想: 每次排除一半搜索空间 (折半查找)
int binarySearch(int arr[], int n, int target) {
    int left = 0, right = n - 1;                // 搜索范围 [left, right]

    while (left <= right) {                      // 搜索空间不为空
        int mid = left + (right - left) / 2;     // 中间位置 (防溢出)

        if (arr[mid] == target) {                 // 找到目标值
            return mid;
        } else if (arr[mid] < target) {          // 目标在右半部分
            left = mid + 1;                      // 缩小范围 (缩小搜索空间)
        } else {                                  // 目标在左半部分
            right = mid - 1;                     // 缩小范围
        }
    }

    return -1;                                   // 未找到
}
```

核心思路：在有序数组中，通过不断将搜索区间减半来快速定位目标值，每次比较中间元素后排除一半的可能性。

**教学重点：** 重点展示"每次排除一半搜索空间"的核心思想，指针移动过程要清晰。

### 6. 深度优先搜索 DFS

**可视化引导：**
```
可视化方式：用树形结构展示图的搜索过程，节点用圆形表示，边用线段连接。
- 节点颜色状态：白色（未访问）、蓝色（正在访问/在栈中）、绿色（已访问/已回溯）
- 当前路径：从根到当前节点的路径用粗线或高亮色标记
- 入栈动画：节点从白色变为蓝色时有一个脉冲/放大效果
- 回溯动画：节点从蓝色变为绿色，路径高亮退回上一层
- 递归深度：可在侧边显示当前递归深度或栈的内容
- 邻居发现：当前节点的未访问邻居用虚线圈标记，提示"接下来要访问"
- 步骤拆分：访问节点 → 标记已访问 → 选择邻居 → 递归进入 → 回溯，每步清晰
- 输入：用户可选择不同的起始节点
- 图结构：使用一个预设的简单无向图（5-8个节点），在页面上展示邻接关系
- 难度标签：基础
```

**参考代码：**

```cpp
// 深度优先搜索 - Depth First Search
// 使用邻接表(邻接矩阵 adj[][] 存储图
// visited[] 记录节点是否已访问
int adj[10][10];  // 邻接矩阵 adjacency matrix (最多10个节点)
int visited[10];

void dfs(int node, int n) {                     // node: 当前节点, n: 节点总数
    visited[node] = 1;                           // 标记为已访问 (mark as visited)
    // 处理当前节点...

    for (int i = 0; i < n; i++) {                // 遍历所有邻居
        if (adj[node][i] == 1 && !visited[i]) {   // 有边且未访问 (相邻且未访问)
            dfs(i, n);                           // 递归访问 (recursion 递归)
        }
    }

    // 回溯 backtrack: 从节点 node 退出, 回到上一层
}
```

核心思路：沿着一条路径尽可能深入，走到尽头后回溯（backtrack）到上一个分支点，继续探索其他路径，属于递归（recursion）的典型应用。

### 7. 广度优先搜索 BFS

**可视化引导：**
```
可视化方式：和DFS使用相同的树形/图结构表示，保持搜索类视觉一致性。
- 节点颜色状态：白色（未访问）、橙色（在队列中）、绿色（已出队/已访问）
- 队列可视化：在页面右侧或下方显示一个队列，实时展示入队/出队过程
- 层级展示：同一层的节点用相同的浅色背景标记（第1层浅蓝、第2层浅紫...）
- 访问动画：节点从白色变为橙色（入队），再变为绿色（出队并访问）
- 邻居发现：当前节点的未访问邻居逐个变为橙色入队
- 步骤拆分：出队当前节点 → 访问并标记 → 未访问邻居入队 → 队列更新
- 难度标签：基础
```

**参考代码：**

```cpp
// 广度优先搜索 - Breadth First Search
// 使用邻接矩阵 + 队列(queue)实现
// visited[] 记录节点是否已访问
int adj[10][10];
int visited[10];
int queue[100];   // 模拟队列
int head = 0, tail = 0;

void bfs(int start, int n) {                    // start: 起始节点
    visited[start] = 1;                          // 标记起始节点已访问
    queue[tail++] = start;                       // 起始节点入队 (enqueue)

    while (head < tail) {                        // 队列不为空 (队列非空)
        int cur = queue[head++];                 // 出队 (dequeue)

        for (int i = 0; i < n; i++) {            // 遍历邻居
            if (adj[cur][i] == 1 && !visited[i]) {
                visited[i] = 1;                  // 标记已访问
                queue[tail++] = i;              // 入队
            }
        }
    }
}
```

核心思路：借助队列（queue）逐层扩展，先访问离起点近的所有节点，再访问更远的节点，适合求无权图的最短路径。

## 第三批：DP补充（dp/）

### 8. 0-1背包问题 0-1 Knapsack

**可视化引导：**
```
可视化方式：用二维表格展示DP矩阵，和LCS保持一致的表格风格。
- 行：物品编号（1~n），列：背包容量（0~W）
- 当前单元格：高亮为金色，显示正在计算的格子
- 物品信息：在表格上方或左侧显示当前物品的重量和价值
- 决策展示：比较"选"和"不选"两种情况时，用两种颜色标注（选=绿色，不选=蓝色）
- 参考单元格：高亮当前决策所依赖的左上方单元格 dp[i-1][j] 和 dp[i-1][j-wi]
- 已填充：计算完成的格子显示数值，未计算显示空
- 最终回溯：填写完成后，从dp[n][W]回溯标记选中的物品（高亮路径）
- 输入：用户可自定义物品数量、重量和价值数组、背包容量
- 难度标签：基础
```

**参考代码：**

```cpp
// 0-1背包问题 - 0-1 Knapsack Problem
// dp[i][j]: 前i个物品、背包容量为j时的最大价值 (maximum value)
// 状态转移方程: dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])
int dp[105][10005];  // dp数组
int w[105];           // 重量 weight
int v[105];           // 价值 value

int knapsack(int n, int W) {                     // n: 物品数, W: 背包容量
    // 边界条件 (boundary): 容量为0或物品数为0时, 价值为0
    for (int i = 0; i <= n; i++)
        dp[i][0] = 0;
    for (int j = 0; j <= W; j++)
        dp[0][j] = 0;

    for (int i = 1; i <= n; i++) {               // 枚举每个物品
        for (int j = 1; j <= W; j++) {           // 枚举每个容量
            if (j >= w[i]) {
                // 选或不选, 取最大值 (max 不选, 选)
                dp[i][j] = dp[i-1][j];
                if (dp[i-1][j-w[i]] + v[i] > dp[i][j])
                    dp[i][j] = dp[i-1][j-w[i]] + v[i];
            } else {
                dp[i][j] = dp[i-1][j];            // 放不下, 只能不选
            }
        }
    }

    return dp[n][W];                              // 返回最大价值
}
```

核心思路：动态规划（dynamic programming）经典问题——对每个物品做出"选"或"不选"的决策，通过状态转移方程（state transition equation）逐步求解最优值。

### 9. 最长递增子序列 LIS

**可视化引导：**
```
可视化方式：用一维数组展示原始序列，下方用一维数组展示dp值，和台阶问题保持一致的风格。
- 原始序列：水平排列，每个元素是一个卡片，显示数值和索引
- dp数组：在原始序列下方对应显示每个位置的最长递增子序列长度
- 当前计算位置：原始序列和dp数组中对应元素同时高亮为橙色
- 回溯来源：用箭头或连线指向 dp[i] 的来源（dp[j] 最大的那个 j），线条颜色为蓝色
- 最终路径：完成后，回溯出的最长递增子序列元素高亮为绿色，用箭头串联
- 输入：用户可自定义序列（逗号分隔的数字）
- 难度标签：基础
```

**参考代码：**

```cpp
// 最长递增子序列 - Longest Increasing Subsequence
// dp[i]: 以 arr[i] 结尾的最长递增子序列长度
// 状态转移方程: dp[i] = max(dp[j] + 1), 其中 j < i 且 arr[j] < arr[i]
int lis(int arr[], int n) {
    int dp[105];
    int ans = 1;

    for (int i = 0; i < n; i++) {               // 枚举每个位置
        dp[i] = 1;                               // 至少包含自身
        for (int j = 0; j < i; j++) {           // 向前查找更小的元素
            if (arr[j] < arr[i]) {               // 递增条件 (严格递增)
                if (dp[j] + 1 > dp[i])
                    dp[i] = dp[j] + 1;           // 更新最优解
            }
        }
        if (dp[i] > ans) ans = dp[i];             // 更新全局最优
    }

    return ans;                                   // 最长递增子序列长度
}
```

核心思路：动态规划——定义 dp[i] 为以第 i 个元素结尾的最长递增子序列长度，通过枚举前面的所有更小元素来转移状态。

### 10. 完全背包问题 Complete Knapsack

**可视化引导：**
```
可视化方式：和0-1背包保持一致的二维表格风格，让学生容易对比两种背包问题的差异。
- 表格结构同0-1背包：行=物品，列=容量
- 关键差异高亮：在决策时，如果选择当前物品，高亮的是同一行的 dp[i][j-wi]（而非上一行的 dp[i-1][j-wi]），用红色边框标注这个差异
- 物品可重复：在物品信息区标注"当前物品可以使用多次"
- 其余交互和0-01背包一致：当前单元格金色高亮、参考单元格标注、最终回溯
- 输入：用户可自定义物品和背包容量
- 难度标签：进阶
```

**参考代码：**

```cpp
// 完全背包问题 - Complete Knapsack Problem
// 与0-1背包的区别: 每个物品可以重复选取 (unlimited / 可重复)
// 状态转移方程: dp[i][j] = max(dp[i-1][j], dp[i][j-w[i]] + v[i])
// 注意: 第二项是 dp[i][j-w[i]], 不是 dp[i-1][j-w[i]] (与0-1背包的唯一区别)
int dp[105][10005];
int w[105];           // 重量
int v[105];           // 价值

int completeKnapsack(int n, int W) {
    for (int i = 0; i <= n; i++)
        dp[i][0] = 0;
    for (int j = 0; j <= W; j++)
        dp[0][j] = 0;

    for (int i = 1; i <= n; i++) {               // 枚举每个物品
        for (int j = 1; j <= W; j++) {           // 枚举容量 (正序遍历)
            if (j >= w[i]) {
                dp[i][j] = dp[i-1][j];            // 不选当前物品
                if (dp[i][j-w[i]] + v[i] > dp[i][j])
                    dp[i][j] = dp[i][j-w[i]] + v[i]; // 选当前物品(可重复选)
            } else {
                dp[i][j] = dp[i-1][j];
            }
        }
    }

    return dp[n][W];
}
```

核心思路：与 0-1 背包的唯一区别在于状态转移方程的第二项使用 dp[i][j-w[i]]（当前行）而非 dp[i-1][j-w[i]]（上一行），体现了物品可重复选取的特性。

**教学重点：** 核心是让学生对比 0-1 背包和完全背包在递推公式上的唯一区别——dp[i-1][j-wi] vs dp[i][j-wi]。
